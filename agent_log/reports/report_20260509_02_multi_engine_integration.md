# 작업 보고서 (2026-05-09) - Multi-Engine 정규식 통합 및 안정화

## 1. 작업 개요
`shield-str` 라이브러리에 **PCRE2** 엔진을 추가로 통합하고, 기존 **RE2** 및 **Standard Library (std::regex)** 엔진과의 호환성을 유지하면서 전체적인 아키텍처를 고도화하였습니다. 특히 멀티 엔진 환경에서의 빌드 안정성과 런타임 성능, 그리고 스레드 안전성을 중점적으로 개선하였습니다.

## 2. 주요 변경 사항

### 2.1. PCRE2 엔진 통합
- `PCRE2Wrapper` 클래스 구현: Unicode 지원, JIT 컴파일 최적화, 캡처 그룹 추출 기능 포함.
- CMake 옵션(`-DSHIELD_REGEX_ENGINE=PCRE2`)을 통한 엔진 선택 기능 추가.

### 2.2. 아키텍처 통합 및 유연성 확보
- `PatternMatcher`를 리팩토링하여 RE2, PCRE2, STD 엔진을 단일 인터페이스(`apply`)로 처리하도록 개선.
- `match_groups` 추상화 패턴을 도입하여 엔진별 캡처 그룹 처리 로직의 파편화를 방지.
- `RuleSet` 클래스를 통해 불변(Immutable) 규칙셋 관리 및 원자적(Atomic) 업데이트 구현.

### 2.3. 안정성 및 버그 수정
- **JSON Unescaping 개선**: `simdjson` 및 표준 라이브러리 환경 모두에서 정규식 패턴 내의 이스케이프 문자(`\\s`, `\\d` 등)가 올바르게 컴파일되도록 커스텀 unescape 로직 구현.
- **ThreadSafetyTest 복구**: 동시성 테스트 중 발생하던 Race Condition 및 결과 검증 로직을 개선하여 모든 엔진에서 테스트 통과 확인.
- **RE2 빌드 회귀(Regression) 해결**: `RE2::FindAndConsumeN`과 `RE2::Match`를 활용하여 동적 캡처 그룹 처리 시 발생하던 템플릿 타입 오류 해결.

### 2.4. 의존성 최적화 (Standard-Only Mode)
- 외부 라이브러리(simdjson, RE2 등) 없이 C++ 표준 라이브러리만으로 동작 가능한 모드(`-DSHIELD_USE_STD_ONLY=ON`) 지원 강화.
- 가벼운 JSON 수동 파싱 로직을 `RuleManager`에 포함하여 이식성 증대.

- **성능 시각화**: 각 엔진별 벤치마크 데이터를 수집하여 시각화 대시보드(GitHub Pages) 구축 예정.

## 5. 제안 사항
- 프로젝트 일관성을 위해 모든 정규식 패턴은 PCRE/RE2 공통 문법을 사용하도록 가이드라인 설정을 권장합니다.
- 성능이 중요한 환경에서는 PCRE2(JIT) 또는 RE2 엔진 사용을 권장하며, 배포 편의성이 중요한 환경에서는 STD 모드를 사용하도록 구성하였습니다.
